### 2.2 设计原则

| 原则 | 说明 |
|------|------|
| **单一职责** | 每个类只负责一件事 |
| **依赖倒置** | 核心逻辑不依赖 UI，通过信号/槽通信 |
| **动画解耦** | 动画通过 manifest.json 配置清单描述，引擎动态加载 |
| **AppContext 集中管理** | 所有管理器单例通过 DI 容器统一持有和访问 |

## 4. 核心类设计

### 4.1 Application — 应用生命周期

| 属性/方法 | 说明 |
|-----------|------|
| `run()` | 初始化顺序：initContext → loadSettings → setTheme → registerQmlTypes → initQmlEngine → connectSignals → initTray → start |
| `m_context` | AppContext 单例，持有所有管理器 |
| `m_bridge` | QmlBridge 实例，暴露给 QML |
| `m_qmlEngine` | QQmlApplicationEngine 实例 |

**特殊行为**：
- `setQuitOnLastWindowClosed(false)` — 窗口关闭不退出应用
- `QML_DISABLE_DISK_CACHE=1` — 禁用 QML 磁盘缓存，确保源码更新后加载最新文件
- 根据系统主题色设置 `QT_QUICK_CONTROLS_DARK_THEME`

### 4.2 AppContext — DI 容器

持有并管理所有核心单例：`ReminderEngine`、`AnimationManager`、`AnimationLoader`、`SettingsManager`、`DrinkTracker`、`QuoteManager`、`SoundManager`、`TrayManager`。

### 4.3 ReminderEngine — 提醒引擎

| 属性/方法 | 说明 |
|-----------|------|
| `start()` | 启动定时器（暂停状态下不启动） |
| `pause(int minutes)` | 停止主定时器，启动暂停定时器（到期自动恢复） |
| `resume()` | 手动恢复：停止暂停定时器，重启主定时器 |
| `triggerNow()` | 立即触发一次提醒，不等待定时器 |
| `m_timer` | 主提醒定时器 |
| `m_pauseTimer` | 暂停定时器（SingleShot，到期触发自动恢复） |

**状态机**：

```
         start()
  [IDLE] ──────────► [RUNNING]
                       │  pause(N)
                       ▼
                    [PAUSED] ── resume() / 到期自动 ──► [RUNNING]
                       │
                       │ stop()
                       ▼
                    [IDLE]
```

### 4.4 AnimationManager — 动画管理器

| 属性/方法 | 说明 |
|-----------|------|
| `play()` | 根据当前主题加载动画描述并发送 playRequested 信号 |
| `stop()` | 停止播放 |
| `setCurrentTheme(id)` | 切换主题（先验证有效性） |
| `availableThemes` | 返回所有可用主题 ID 列表 |

### 4.5 DrinkTracker — 饮水记录

| 属性/方法 | 说明 |
|-----------|------|
| `recordDrink()` | 记录一次饮水（QDate → count 递增） |
| `clearHistory()` | 清除所有历史记录，归零计数 |
| `todayCount` | 今日饮水杯数 |
| `weekCount` | 本周饮水杯数（周一至当天） |
| `m_dailyRecords` | `QMap<QDate, int>` 存储每日杯数 |
| 保留策略 | 仅保留最近 30 天记录 |

### 4.6 SettingsManager — 配置管理

| 属性 | 默认值 | 说明 |
|------|--------|------|
| `intervalMinutes` | 60 | 提醒间隔（分钟） |
| `animationTheme` | "skeleton" → 自动迁移 | 当前动画主题 ID |
| `animationDuration` | 6 | 动画持续时长（秒） |
| `autoStart` | false | 开机自启 |

存储位置：`QSettings("deepin", "deepin-water-reminder")`

### 4.7 TrayManager — 系统托盘

| 方法 | 说明 |
|------|------|
| `setDefaultIcon()` | 设置默认托盘图标 |
| `setRemindingIcon()` | 设置提醒中图标（未使用，预留） |
| `setPausedIcon()` | 设置暂停图标 + 更新 m_paused 状态 |
| `setPaused(bool)` | 公开接口，统一设置暂停状态和图标 |
| `showContextMenu()` | 构建右键菜单 |

**右键菜单行为**：
- 未暂停时：显示完整菜单，「暂停提醒」为菜单项，点击后暂停所有提醒
- 已暂停时：简化菜单，仅显示「▸ 立即喝一杯」和「▶ 恢复提醒」

### 4.8 QmlBridge — C++ ↔ QML 桥接

将 C++ 核心对象暴露给 QML 端。

| QML 属性 | C++ 来源 |
|-----------|----------|
| `bridge.settings` | SettingsManager 属性映射为 QVariantMap |
| `bridge.currentTheme` | AnimationManager::currentTheme() |
| `bridge.availableThemes` | AnimationManager::availableThemes() |
| `bridge.todayDrinkCount` | DrinkTracker::todayCount() |
| `bridge.weekDrinkCount` | DrinkTracker::weekCount() |
| `bridge.paused` | ReminderEngine 暂停状态 |

| QML 调用 | C++ slot |
|-----------|----------|
| `bridge.recordDrink()` | DrinkTracker::recordDrink() |
| `bridge.clearDrinkHistory()` | DrinkTracker::clearHistory() |
| `bridge.saveSettings({...})` | SettingsManager 各 setter + save() |
| `bridge.playAnimation()` | AnimationManager::play() |
| `bridge.setTheme(id)` | AnimationManager + SettingsManager |
| `bridge.triggerReminder()` | ReminderEngine::triggerNow() |
| `bridge.pauseReminder(min)` | ReminderEngine::pause(min) |
| `bridge.resumeReminder()` | ReminderEngine::resume() |

### 4.9 QuoteManager — 趣味话语管理器

| 属性/方法 | 说明 |
|-----------|------|
| `loadQuotes()` | 加载 18 条硬编码趣味饮水话语 |
| `randomQuote()` | 使用 Fischer-Yates 洗牌算法随机返回一条话语，一轮内不重复 |
| `rebuildShuffleOrder()` | 重建洗牌顺序，确保每轮所有话语都被使用后才重新洗牌 |
| `m_quotes` | `QVector<QString>` 内置话语列表 |
| `m_shuffleOrder` | `QVector<int>` 当前轮换顺序索引 |

**特点**：完全硬编码，不依赖外部文件，不重复轮换保证用户体验。

### 4.10 SoundManager — 音效管理器

| 属性/方法 | 说明 |
|-----------|------|
| `playDrinkReminder()` | 播放喝水提醒音效，无音源或加载失败时静默降级 |
| `loadFallbackSource()` | 加载兜底音源 |
| `m_effect` | `QSoundEffect*` 音效播放器 |
| `m_loaded` | 音源是否已加载 |
| `m_available` | 音效是否可用（硬件/音源检测） |

**特点**：使用 QSoundEffect 低延迟音频播放，无音源时不崩溃、静默降级。

## 5. QML 界面设计

### 5.1 main.qml — 隐藏宿主窗口

1×1 像素、不可见、SplashScreen + FramelessWindowHint。仅作为 QQmlApplicationEngine 的宿主。

动态创建 TransparentOverlay（懒加载），通过 Loader 懒加载 SettingsPanel / DrinkRecordPanel / AboutPanel。

### 5.2 TransparentOverlay.qml — 动画覆盖层

- 全屏透明窗口（FramelessWindowHint + WindowStaysOnTopHint）
- `showOverlay(animPath)`：定位到主屏幕、设置尺寸、加载动画、启动自动隐藏定时器
- `hideOverlay()`：隐藏窗口、停止定时器、卸载动画
- `shake(intensity)`：动画调用，产生窗口抖动效果（雷暴主题使用）
- 点击动画区域 → `bridge.onUserDismissed()`（记录饮水 + 关闭）
- 自动隐藏定时器根据 `bridge.settings.animationDuration` 设置

### 5.3 AnimatedText.qml — 趣味话语逐字动画（v1.1 新增）

- 将文字逐字符拆解为独立 `Text` 元素，通过 `Repeater` + `ListModel` 渲染
- 支持 3 种动画风格：四面八方飞入（style=0）、从下往上弹入（style=1）、缩放旋转渐入（style=2）
- 每个字符独立 `SequentialAnimation`：延迟入场 → 飞入/弹入到位 → 循环微浮动
- 中文字符宽度为 `fontSize * 1.0`，英文字符宽度为 `fontSize * 0.62`
- 字符描边（outlineColor）确保在任何背景上清晰可见
- 入场完成后持续上下微浮动（2000ms 周期，InOutSine 缓动）

### 5.4 动画实现规范

所有动画使用展开式多行 QML 格式（避免单行压缩写法导致替换/解析错误），每个 GradientStop 属性独占一行。

动画根元素为 `Rectangle { color: "transparent"; anchors.fill: parent }`，内部通过 Repeater + SequentialAnimation/ParallelAnimation + NumberAnimation 实现动态效果。

## 6. 信号流

### 6.1 定时提醒流程

```
QTimer timeout → ReminderEngine.onTimeout()
  → emit reminderTriggered()
  → QmlBridge 连接槽：获取当前主题的动画路径
  → emit overlayRequested(path)
  → main.qml 接收：创建/获取 OverlayWindow
  → overlay.showOverlay(path)
    → 加载动画 → 显示 → 启动自动隐藏定时器
    → 用户点击 / 定时器到期
    → hideOverlay()
    → 卸载动画
```

### 6.2 用户饮水记录流程

```
用户点击动画 → QmlBridge.onUserDismissed()
  → DrinkTracker.recordDrink()
  → emit todayCountChanged() / weekCountChanged()
  → QmlBridge 转发 → QML 绑定更新
  → AnimationManager.stop()
  → emit overlayHideRequested()
  → overlay.hideOverlay()
```

### 6.3 暂停/恢复流程

```
用户选择暂停 → TrayManager.emit pauseRequested(N)
  → ReminderEngine.pause(N)
    → 停止主定时器，启动暂停定时器
    → emit pausedStateChanged(true)
    → Application 连接：trayManager.setPaused(true) → 更新图标
    → 定时器到期 → 自动 resume

用户选择恢复 → TrayManager.emit resumeRequested()
  → ReminderEngine.resume()
    → 停止暂停定时器，重启主定时器
    → emit pausedStateChanged(false)
    → Application 连接：trayManager.setPaused(false) → 恢复默认图标
```
