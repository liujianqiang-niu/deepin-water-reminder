## 2. 项目背景与目标

### 2.1 背景

长时间办公/编码人群普遍存在饮水不足的问题。市面上已有一些喝水提醒工具，但多数交互生硬（弹窗式通知），缺乏趣味性，用户容易忽略或直接关闭。本项目旨在打造一款 **趣味化、非侵入式** 的桌面喝水提醒应用，通过生动的桌面动画吸引用户注意，提升用户体验和饮水意愿。

### 2.2 目标

1. **沉浸式提醒**：以多层次全屏动画（海洋、星空、极光、雷暴、火焰）形式提醒用户喝水，震撼且有舒适感。
2. **非侵入式**：动画窗口透明无边框，不抢夺焦点，不影响用户当前工作。
3. **轻量稳定**：资源占用低，在后台静默运行，定时触发。
4. **灵活控制**：暂停/恢复、主题切换、间隔自定义、饮水记录。

## 4. 非功能需求

### 4.1 性能要求

| 指标 | 要求 |
|------|------|
| 内存占用（空闲） | ≤ 50 MB |
| 内存占用（动画播放） | ≤ 120 MB |
| CPU 占用（空闲） | ≤ 0.5% |
| CPU 占用（动画播放） | ≤ 8%（普通办公机） |
| 启动时间 | ≤ 2 秒 |

### 4.2 兼容性要求

| 项目 | 要求 |
|------|------|
| 操作系统 | Linux（Deepin 20+/UOS 20+/Ubuntu 20.04+/Fedora 36+） |
| 显示服务器 | X11 为主要目标，Wayland 兼容 |
| Qt 版本 | Qt 6.5+ |
| 屏幕分辨率 | 1366×768 及以上，支持高分屏缩放 |

### 4.3 可用性要求

- 动画窗口不抢占键盘焦点
- 点击动画区域可关闭动画并记录「已喝」
- 系统托盘图标始终可见，三种状态（默认/提醒中/已暂停）切换
- 暂停后托盘菜单简化，突出恢复入口

### 4.4 可维护性要求

- 动画资源与业务逻辑解耦，通过 manifest.json 配置清单管理
- 代码模块化，核心逻辑与 UI 分离（C++ 核心 + QML 界面 + QmlBridge 桥接）
- 完善的日志记录（提醒触发、动画播放、用户操作、配置变更）
- QML 禁用磁盘缓存（`QML_DISABLE_DISK_CACHE=1`），避免源码更新后加载旧缓存

### 5.1 主界面

本应用 **无传统主窗口**，以系统托盘 + 设置界面 + 动画弹出层 + 饮水记录面板 + 关于面板构成。主窗口为 1×1 像素的隐藏窗口，仅作为 QML 引擎宿主。

### 5.4 动画弹出层

- **窗口尺寸**：全屏覆盖（与主屏幕等大）
- **背景**：完全透明（alpha = 0），仅有动画元素可见
- **层级**：置顶（`WindowStaysOnTopHint`），不抢焦点
- **交互**：点击任意位置 → 动画消失并记录「已喝」；倒计时结束 → 自动消失
- **展示时长**：用户可配置（默认 8 秒）

## 6. 约束与假设

### 6.1 技术约束

1. 使用 Qt 6 + QML 开发。
2. Linux 透明窗口在 X11/Wayland 下行为有差异，需分别处理。
3. 系统托盘在 Wayland 下可能需要 SNI（StatusNotifierItem）协议支持。

### 6.2 假设

1. 用户桌面环境支持系统托盘（如 Deepin Dock、GNOME Top Bar、KDE Panel）。
2. 用户已安装 Qt 6 运行库。
3. 动画通过 QML 原生图形实现，无需第三方动画库。

## 8. 风险与对策

| 风险 | 影响 | 对策 |
|------|------|------|
| Wayland 下透明窗口支持不完善 | 动画无法透明显示 | 优先支持 X11，Wayland 作为降级方案 |
| 系统托盘 API 差异 | 部分桌面环境托盘不可用 | 依赖 QSystemTrayIcon 跨平台实现 |
| QML 粒子动画性能 | 大量粒子可能影响低端机 | 限制粒子数量，优先使用简单图形而非复杂路径 |
